# Contributing

欢迎基于这个仓库扩展多 Agent 研发团队配置，但请保持以下约束：

## 提交原则

- 不要绕过 [`AGENTS.md`](./AGENTS.md) 中定义的门禁语义
- 不要随意新增同义 artifact 标签
- 修改角色职责时，同时更新文档与对应的 `.codex/agents/*.toml`
- 评审、安全、审批类角色默认保持只读

## 建议提交流程

1. 创建 issue，说明新增角色、变更流程或模板改动的目标。
2. 修改相应配置与文档。
3. 在 PR 描述中说明：
   - 影响的角色
   - 影响的 artifact
   - 是否改变门禁逻辑
   - 是否需要迁移已有项目
4. 由维护者评审后合并。

## Pull Request 检查项

- [ ] 文档与 agent 配置保持一致
- [ ] 新增角色定义了输入、输出、权限、交接对象
- [ ] 变更没有破坏现有 artifact 标签体系
- [ ] 示例流程仍然闭环

## 风格约束

- 优先使用清晰、可执行的规则，而不是抽象口号
- 角色边界要可落地，避免职责重叠
- 模板内容应服务于交付，不要堆砌冗余字段
