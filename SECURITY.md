# Security Policy

## Supported Scope

当前安全策略主要覆盖以下内容：

- `AGENTS.md` 中定义的角色门禁与交接规则
- `.codex/agents/*.toml` 中的 agent 权限、职责与输入输出约束
- 文档、模板与协作流程中的安全边界定义

## Reporting a Vulnerability

如果你发现以下类型的问题，请私下联系维护者，而不要直接公开披露：

- 角色权限越权
- 审批门禁可被绕过
- 安全审计角色被错误配置为可写
- Prompt / Agent 指令存在明显注入或权限穿透风险
- 模板或流程会导致敏感数据泄露

建议报告内容包括：

- 问题标题
- 影响范围
- 复现步骤
- 预期行为与实际行为
- 风险等级判断
- 修复建议

## Response Expectations

维护者收到报告后，通常应按以下顺序处理：

1. 确认问题是否可复现
2. 判断影响范围与严重级别
3. 制定修复方案
4. 在修复完成后再决定公开披露方式

## Disclosure Guidance

在问题修复前，请不要公开：

- 可直接利用的绕过方法
- 敏感配置细节
- 可能导致误用的 exploit 描述
