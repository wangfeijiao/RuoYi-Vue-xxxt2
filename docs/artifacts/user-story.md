## Epic 1: IT资源管理

### US-IT-01 资源登记与台账建立

- 作为 IT资源管理员，我希望登记服务器、虚拟机、存储设备、中间件和软件许可证，并维护其配置、责任人、部门和关联关系，以便形成准确可追溯的资源台账。
- 验收条件
1. 支持按资源类型录入差异化属性。
2. 每个资源具备唯一标识和状态字段。
3. 可记录资源间依赖关系及所属项目/应用关系。
4. 资源信息变更后保留更新时间和操作记录。

### US-IT-02 项目资源申请与分配

- 作为项目经理，我希望按资源类型提交 IT 资源申请并关联项目，以便快速获得项目建设所需环境。
- 验收条件
1. 申请表单根据资源类型动态展示必填项。
2. 申请时必须可选择关联项目。
3. 资源管理员可审核并执行“分配现有资源”或“新建资源”。
4. 审批通过后自动生成待交付任务并通知相关运维人员。

### US-IT-03 资源交付、变更与回收

- 作为 IT运维人员，我希望在交付、扩容、续期、回收时记录执行结果和部署信息，以便资源生命周期完整留痕。
- 验收条件
1. 交付环节可填写部署信息、交付时间、执行人和结果。
2. 变更申请必须绑定已有资源 ID，并自动带出原配置。
3. 回收完成后资源状态更新为“已回收”，并释放对应资源配额。
4. 整个生命周期可在台账中追踪申请、审批、执行和状态变化。

## Epic 2: 应用管理

### US-APP-01 应用台账登记

- 作为应用系统管理员，我希望登记每个应用系统的基础信息、责任人、归属单位和管理团队负责人，以便明确应用责任边界。
- 验收条件
1. 应用台账包含名称、版本、开发商、技术栈、访问地址、系统负责人等基础信息。
2. 可维护业主方负责人、开发单位负责人及各管理团队责任人。
3. 支持区分自建、集团统建、交通部统建等分类。

### US-APP-02 应用依赖关系管理

- 作为应用系统管理员，我希望维护应用与服务器、数据库、API、项目之间的依赖关系，以便掌握应用上下游影响范围。
- 验收条件
1. 应用详情可关联底层 IT 资源、数据库、API 和所属项目。
2. 能查看应用与资源的依赖关系清单。
3. 依赖对象变化后，应用侧关联信息同步更新。

### US-APP-03 应用运行监测与通知

- 作为应用负责人，我希望看到应用相关的 IT 资源和网络状态，并在异常时收到通知，以便及时组织处理。
- 验收条件
1. 应用详情可展示底层资源状态和网络状态。
2. 当底层状态异常时，可通知对应应用责任人。
3. 支持按应用分类、状态、单位进行统计分析。

## Epic 3: 信息化项目管理

### US-PROJ-01 项目登记与模板装配

- 作为项目经理，我希望登记新建或存量项目并根据项目类型自动生成标准文档目录，以便快速建立规范化项目空间。
- 验收条件
1. 项目登记可录入项目类型、状态、业主方负责人、开发单位负责人及管理团队责任人。
2. 选择项目类型后自动生成对应文档目录模板。
3. 项目经理可补充非标准目录。

### US-PROJ-02 项目过程资料管理

- 作为资料管理员，我希望督导项目各阶段按目录上传关键资料，以便形成从立项到验收的全过程归档基础。
- 验收条件
1. 项目空间支持按目录上传和查看过程资料。
2. 资料可按阶段、目录、更新时间追踪缺失情况。
3. 对不同成员支持查看、编辑、下载、删除等细粒度权限控制。

### US-PROJ-03 项目验收申请与审核

- 作为项目经理，我希望在项目结束时提交验收申请，并由 IT、数据、安全、资料等相关角色审核，以便完成规范结项。
- 验收条件
1. 验收申请必须覆盖 IT管理、数据管理、安全管理、过程资料等审核维度。
2. 审核人可分别给出通过/退回结论和意见。
3. 项目状态可根据审核进度变化为待验收、审核中、已验收、已结项。
4. 验收通过后可触发后续资源回收和资料归档动作。

## Epic 4: 数据资源管理

### US-DATA-01 数据资源接入与目录建设

- 作为数据资源管理员，我希望通过手工录入或接口对接接入数据资源，并建立标准化目录信息，以便让数据可发现、可治理。
- 验收条件
1. 可登记数据资源的基础信息、内容结构、管理技术属性、安全共享属性。
2. 支持记录数据来源系统、更新频率、数据量、存储位置。
3. 可维护数据字典、元数据、血缘和敏感数据标识。

### US-DATA-02 数据目录浏览与申请

- 作为数据使用者，我希望检索并浏览授权范围内的数据目录，并提交数据使用申请，以便获得业务所需数据。
- 验收条件
1. 目录门户支持按分类、部门、标签等方式浏览和检索。
2. 不同状态的数据资源具备不同可见性和可申请性。
3. 申请时需填写业务需求和使用目的。
4. 审批通过后可查看申请结果和后续数据提供方式。

### US-DATA-03 数据资源维护与变更审批

- 作为数据责任人，我希望维护本人负责的数据资源，并在修改安全等级、共享方式、责任人或结构时走审批流程，以便保证数据治理合规。
- 验收条件
1. 责任人可直接修改简介、更新频率、联系方式等非核心属性。
2. 核心属性变更必须提交变更申请。
3. 系统保留元数据历史版本，可查看快照和版本差异。
4. 能记录审批结论和变更生效状态。

## Epic 5: 网络安全管理

### US-NET-01 网络资源台账与策略管理

- 作为网络资源管理员，我希望维护 IP、VLAN、带宽、设备、端口和安全策略台账，以便统一掌握网络资源和安全边界。
- 验收条件
1. 可登记网络资源基础属性和所属责任信息。
2. 可维护防火墙规则、访问控制列表等策略信息。
3. 可查看网络逻辑拓扑或连接关系描述。

### US-NET-02 网络申请、审核与配置

- 作为项目组成员，我希望按需申请 IP、端口、带宽等网络资源，并由网络管理员审核配置，以便满足项目接入需要。
- 验收条件
1. 申请表单支持按网络资源类型动态展示字段。
2. 审核环节可评估 IP 数量、端口开放、带宽要求和安全合规性。
3. 审批通过后可记录配置执行结果，包括 IP 分配、VLAN 配置、访问权限开通。

### US-NET-03 网络变更、监控与回收

- 作为网络资源管理员，我希望跟踪网络变更、监控资源使用并及时回收释放资源，以便避免冲突、滥用和资源浪费。
- 验收条件
1. 网络变更必须记录变更内容、执行时间和执行人。
2. 可监控使用情况并识别 IP 冲突、带宽滥用等风险。
3. 项目释放网络资源后可完成回收并更新台账。
4. 可维护网络拓扑图和 IP 分配表等文档。

## 跨模块故事

### US-CROSS-01 首页工作台与待办联动

- 作为多角色协同人员，我希望在首页集中查看待办审批、待交付任务、异常告警和消息通知，以便统一处理高优先级工作。
- 验收条件
1. 首页可聚合资源申请、项目验收、数据申请、网络审核等待办。
2. 告警消息可定位到具体资源、应用、项目或数据对象。
3. 用户仅能看到自己有权限处理的待办和消息。

### US-CROSS-02 项目维度统一关联视图

- 作为管理层，我希望从项目视角查看相关 IT 资源、应用、数据资源、网络资源和验收进度，以便了解项目全生命周期投入与状态。
- 验收条件
1. 项目详情可查看关联资源清单和状态概览。
2. 可追踪项目在立项、执行、验收、结项各阶段的关键事项。
3. 统计分析支持按项目维度汇总资源投入和管理状态。
