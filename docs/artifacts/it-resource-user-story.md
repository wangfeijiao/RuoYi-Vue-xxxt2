### 一、史诗划分

1. Epic A：资源台账管理
2. Epic B：资源申请与交付
3. Epic C：资源变更与回收
4. Epic D：查询统计与监控
5. Epic E：权限与审计

### 二、用户故事清单

#### US-01 资源台账登记

角色：IT 资源管理员
场景：需要将新购置或已有资源纳入统一台账。
价值：避免资源散落在 Excel 或个人记录中，保证资源可查。
优先级：P0 / MVP

验收条件：
1. 可以新增资源编码、名称、类型、项目、责任人、维护人、配置摘要。
2. 资源编码唯一。
3. 支持录入 CPU、内存、存储、操作系统、IP、软件版本等基础配置。
4. 保存后资源立即可在台账列表中检索。
5. 系统记录创建人与创建时间。

#### US-02 资源台账编辑

角色：IT 资源管理员
场景：需要修正资源的非核心信息或补全交付信息。
价值：保证台账持续准确。
优先级：P0 / MVP

验收条件：
1. 可编辑维护人、性能摘要、关联说明、备注等信息。
2. 若修改核心字段，应提示应通过变更工单处理。
3. 保存后更新人和更新时间可见。
4. 已回收资源默认不可直接编辑为使用中。

#### US-03 发起资源申请

角色：项目经理
场景：项目建设中需要申请服务器、虚拟机、存储或软件资源。
价值：标准化申请入口，减少线下沟通。
优先级：P0 / MVP

验收条件：
1. 可选择项目并填写申请标题、申请原因、资源类型、期望配置、期望完成时间。
2. 提交后自动生成工单编号。
3. 工单状态默认为 `PENDING`。
4. 工单能关联到项目。
5. 申请人可在工单列表中查看进度。

#### US-04 审批资源申请

角色：IT 资源管理员
场景：收到资源申请后，需要判断是否分配资源。
价值：控制资源使用合理性。
优先级：P0 / MVP

验收条件：
1. 可查看申请原因、项目、期望配置、期望完成时间。
2. 支持通过或拒绝。
3. 拒绝时必须填写意见。
4. 通过后工单进入 `PENDING_EXECUTION`。
5. 审批人和审批时间被记录。

#### US-05 执行资源交付

角色：运维执行人
场景：审批通过后需要完成实际交付。
价值：形成从审批到落账的闭环。
优先级：P0 / MVP

验收条件：
1. 可填写实际交付配置、交付说明、交付时间。
2. 可创建新资源台账或绑定已有空闲资源。
3. 执行完成后工单状态变为 `COMPLETED`。
4. 资源状态更新为 `IN_USE`。
5. 执行人和执行时间被记录。

#### US-06 发起资源变更

角色：项目经理 / IT 资源管理员
场景：资源需要扩容、缩容、软件升级或责任人调整。
价值：避免直接改台账导致无审计轨迹。
优先级：P0 / MVP

验收条件：
1. 必须基于已有资源发起。
2. 系统自动保存当前资源快照。
3. 可填写变更原因和变更项。
4. 审批通过并执行后，台账更新为最新配置。
5. 可追溯变更前后的差异。

#### US-07 发起资源回收

角色：项目经理 / IT 资源管理员
场景：项目结束或资源闲置，需要回收。
价值：避免资源长期挂账占用。
优先级：P0 / MVP

验收条件：
1. 只能对非空闲资源发起回收。
2. 回收工单需填写回收原因。
3. 审批通过后进入待执行。
4. 执行完成后资源状态变为 `RECYCLED` 或 `IDLE`。
5. 若与项目解绑，解绑关系可追溯。

#### US-08 查询资源台账

角色：IT 资源管理员 / 项目经理 / 管理层
场景：需要快速找到某类资源或某项目占用资源。
价值：提升管理效率。
优先级：P0 / MVP

验收条件：
1. 支持按资源编码、名称、IP、类型、状态、项目、责任人筛选。
2. 列表展示关键配置和状态摘要。
3. 支持查看资源详情。
4. 支持分页。
5. 支持导出当前筛选结果。

#### US-09 查询资源工单

角色：申请人 / IT 资源管理员 / 运维执行人
场景：需要追踪申请、审批、执行状态。
价值：减少反复问询。
优先级：P0 / MVP

验收条件：
1. 支持按工单编号、工单类型、状态、项目、申请人筛选。
2. 可查看审批意见和执行结果。
3. 超时工单有明显标识。
4. 不同角色仅可见有权限的数据范围。

#### US-10 查看资源统计

角色：管理层 / IT 资源管理员
场景：需要掌握资源使用情况和待办情况。
价值：支持决策和催办。
优先级：P1 / MVP

验收条件：
1. 展示资源总数、使用中数量、空闲数量、已回收数量。
2. 展示待审批工单数、待执行工单数。
3. 支持按项目和资源类型统计。
4. 统计结果与台账、工单数据一致。

#### US-11 处理审批拒绝

角色：项目经理
场景：资源申请被拒绝，需要知道原因并重新提交。
价值：缩短沟通成本。
优先级：P1 / MVP

验收条件：
1. 在工单详情中可见拒绝理由。
2. 被拒工单不可继续执行。
3. 可根据原工单内容重新发起新申请。

#### US-12 监控异常查看

角色：IT 资源管理员 / 运维执行人
场景：资源运行异常时需要快速识别。
价值：提升资源稳定性。
优先级：P1 / MVP

验收条件：
1. 台账可展示监控状态。
2. 支持筛选告警资源。
3. 资源详情可查看性能摘要或异常摘要。
4. MVP 不要求自动告警同步，但需保留展示位。

### 三、异常与边界类用户故事

#### US-13 防止重复台账

角色：IT 资源管理员
场景：录入重复资源。
价值：保证台账可信。
优先级：P0 / MVP

验收条件：
1. 重复资源编码禁止保存。
2. 运行中资源的 IP 冲突禁止保存。
3. 页面给出明确错误提示。

#### US-14 禁止非法状态操作

角色：系统
场景：对已回收资源发起变更、对未审批工单直接执行。
价值：防止流程失控。
优先级：P0 / MVP

验收条件：
1. 状态不合法时按钮不可用或接口拒绝。
2. 错误信息明确说明原因。
3. 所有非法操作被记录到审计日志。
