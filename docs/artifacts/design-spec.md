## 1. 设计原则

- 延续现有 RuoYi 后台风格，不重做整体主题
- 通过“看板卡片 + 表格 + 详情对话框/分栏表单”完成一期交互
- 重点提升业务信息密度和状态可读性，避免过度装饰

## 2. 页面布局

### Dashboard

- 顶部使用两行卡片
- 第一行 6 张 KPI 卡片
- 第二行左侧待办表格，右侧最近工单/异常列表

### 台账页

- 顶部查询表单
- 中部工具栏
- 下方主表格
- 详情使用对话框或抽屉，不进入复杂多级路由

### 复合业务页

项目、资源、数据资产、网络资源页面使用 `el-tabs` 区分：

- 台账
- 工单/申请
- 版本或验收等次级信息

## 3. 组件规范

- 查询区统一使用 `el-form` + 行内表单
- 数据表统一使用 `el-table`
- 状态统一使用 `el-tag`
- 审批/执行操作使用主按钮 + 二次确认
- JSON/结构化长文本统一使用 `textarea`
- 统计图一期优先使用卡片和简洁条目，不强依赖复杂图表

## 4. 状态视觉规则

- `PENDING` / 待审核：橙色
- `APPROVED` / 已通过：蓝色
- `REJECTED` / 已驳回：红色
- `PENDING_EXECUTION` / 待执行：紫蓝色
- `COMPLETED` / 已完成：绿色
- `ALERT` / 异常：红色
- `IN_USE` / 在用：绿色
- `RECYCLED` / 已回收：灰色

## 5. 关键字段优先级

表格优先展示：

- 编号
- 名称
- 类型/分类
- 状态
- 责任人
- 关联项目
- 更新时间

详情优先展示：

- 责任矩阵
- 业务摘要
- 配置摘要
- 当前状态
- 审批或变更时间线

## 6. 页面清单

- 信息化总览
- 项目管理
- 应用管理
- IT 资源管理
- 数据资源管理
- 网络安全管理

## 7. 响应式要求

- 桌面端以 24 栅格为主
- 窄屏下查询条件自动换行
- 详情弹窗宽度控制在 900px 左右，避免横向滚动
