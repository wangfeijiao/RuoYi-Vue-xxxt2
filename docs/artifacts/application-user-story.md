## Epic 1: 应用台账管理

### US-APP-01 应用台账登记

- 作为应用系统管理员，我希望登记应用系统的基础信息、建设来源、归属单位和访问地址，以便形成统一的应用台账。
- 验收条件
1. 应用台账至少包含应用编码、名称、类型、建设来源、分级分类、运行状态、访问地址、技术栈。
2. 应用编码必须唯一。
3. 应用支持关联所属项目。
4. 保存成功后可在应用列表中检索到该应用。

### US-APP-02 责任矩阵维护

- 作为应用系统管理员，我希望为每个应用维护业主方负责人、开发单位负责人以及统一认证、IT 资源、数据资源、网络安全等团队负责人，以便责任边界清晰。
- 验收条件
1. 应用详情中可维护多类责任角色。
2. 至少一名业务责任人或运维责任人必填。
3. 各责任字段可单独修改并保留更新记录。
4. 责任人信息可用于通知与权限判定。

## Epic 2: 应用依赖关系管理

### US-APP-03 项目关联维护

- 作为应用系统管理员，我希望把应用与所属项目建立关联，以便从项目维度查看应用清单和建设成果。
- 验收条件
1. 应用可选择所属项目。
2. 项目详情中可查看关联应用列表。
3. 已停用或不存在的项目不能继续建立新关联。

### US-APP-04 资源与接口依赖维护

- 作为应用系统管理员，我希望维护应用与服务器、虚拟机、数据库、API、网络资源之间的依赖关系，以便识别应用运行依赖和影响范围。
- 验收条件
1. 应用详情可维护至少四类依赖对象：IT 资源、数据库/API、网络资源、项目。
2. 应用可查看当前依赖清单与依赖摘要。
3. 依赖对象变更后，应用侧展示同步更新。
4. 无结构化依赖时，系统仍要求填写依赖说明。

## Epic 3: 应用运行视图与异常联动

### US-APP-05 应用运行概览查看

- 作为应用运维人员，我希望在应用详情中看到应用状态、底层 IT 资源状态、网络状态和最近异常，以便快速判断系统健康度。
- 验收条件
1. 应用详情展示应用主状态和联动状态摘要。
2. 可查看底层 IT 资源状态汇总。
3. 可查看网络状态汇总。
4. 可查看最近异常时间、异常对象和处理状态。

### US-APP-06 异常通知触达

- 作为业主方负责人或应用运维人员，我希望当底层 IT 资源或网络异常影响应用时收到通知，以便及时响应。
- 验收条件
1. 当关联资源或网络状态异常时，系统可生成应用异常消息。
2. 通知至少支持站内待办，并预留短信渠道。
3. 通知中包含应用名称、影响对象、异常时间、责任人。
4. 通知记录可在应用详情中追踪。

## Epic 4: 分类分级与统计分析

### US-APP-07 分类分级查询

- 作为应用管理员，我希望按照建设来源、分类分级、所属单位、状态筛选应用，以便高效治理台账。
- 验收条件
1. 列表页支持按建设来源筛选，包括自建、集团统建、交通部统建。
2. 支持按分级分类筛选。
3. 支持按所属单位、责任人、运行状态筛选。
4. 筛选结果可分页浏览与导出。

### US-APP-08 管理统计分析

- 作为管理层，我希望查看应用数量、分类分布、异常数量和责任团队分布，以便掌握治理全貌。
- 验收条件
1. 统计页可展示应用总量。
2. 可展示按建设来源、状态、所属单位的分布结果。
3. 可展示异常应用数量。
4. 可展示责任团队分布或风险归属视图。

## Epic 5: 应用域权限管理

### US-APP-09 应用角色权限边界

- 作为系统管理员，我希望给应用运维人员、业主方负责人、管理团队负责人分配不同的查看与操作权限，以便控制数据边界。
- 验收条件
1. 不同角色的应用列表可见范围不同。
2. 不同角色的编辑字段范围不同。
3. 只有授权角色可以处理应用通知或维护责任矩阵。
4. 权限限制时系统明确提示原因。

## Epic 6: 跨模块联动

### US-APP-10 项目视角查看应用

- 作为项目经理，我希望在项目详情中看到关联应用及其运行状态，以便判断项目建设成果和运维风险。
- 验收条件
1. 项目详情可查看关联应用列表。
2. 每个应用展示状态、责任人和关键依赖摘要。
3. 异常应用在项目视角下有明显标识。

### US-APP-11 应用影响范围追踪

- 作为网络安全团队负责人，我希望从应用视角看到关联资源与网络对象，以便在异常时快速识别影响范围。
- 验收条件
1. 应用详情可追踪到受影响的底层对象。
2. 可查看对应责任团队。
3. 可查看最近处理记录或备注。
