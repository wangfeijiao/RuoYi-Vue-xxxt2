## 1. 信息架构

### 1.1 一级导航

- 首页工作台
- IT资源管理
- 应用管理
- 信息化项目管理
- 数据资源管理
- 网络安全管理
- 公共能力入口：消息中心、我的待办、统计分析

### 1.2 首页工作台

- 顶部：全局检索、消息提醒、当前用户角色切换提示。
- 第一屏：待办审批卡片，按“资源申请”“项目验收”“数据申请”“网络审核”分类展示。
- 第二屏：运行告警与异常消息，展示资源、应用、网络相关告警。
- 第三屏：管理看板，展示资源总量、项目建设数、数据资源数、网络资源使用概览。
- 快捷入口：新建项目、发起资源申请、发起数据申请、查看项目验收任务。

## 2. 页面原型说明

### 2.1 IT资源管理

#### IT资源台账列表页

- 左侧为筛选区：资源类型、状态、所属项目、所属部门、责任人、告警状态。
- 中部为列表区：资源名称、类型、配置摘要、状态、责任人、最近更新时间、关联项目。
- 顶部操作：新增登记、导入、导出、发起申请、批量回收。
- 列表行操作：查看详情、发起变更、发起回收、查看台账。

#### IT资源详情页

- 基础信息区：配置属性、管理属性、关系属性。
- 生命周期区：申请、审批、交付、变更、回收时间线。
- 关联区：关联项目、关联应用、关联网络资源。
- 监控区：运行状态、性能指标占位、最近告警。
- 操作区：变更、续期、终止/回收。

#### IT资源申请页

- 资源类型切换后动态改变表单。
- 表单区包含项目关联、申请原因、配置需求、期望交付时间。
- 右侧显示审批流预览和关联项目基础信息。
- 提交后进入审批状态页。

### 2.2 应用管理

#### 应用台账列表页

- 筛选项：应用分类、所属单位、系统状态、负责人、管理团队。
- 列表项：应用名称、版本、分类、责任人、关联项目数、运行状态。
- 顶部操作：新增应用、导入台账、查看统计。

#### 应用详情页

- 基础信息标签页：应用基础属性与责任人。
- 依赖关系标签页：服务器、数据库、API、网络资源、关联项目。
- 运行概览标签页：底层资源状态、网络状态、异常消息。
- 统计分析标签页：应用数量、分类分布、运行健康度。

### 2.3 信息化项目管理

#### 项目列表页

- 筛选项：项目类型、项目状态、所属单位、项目经理、验收状态。
- 列表项：项目名称、类型、当前阶段、责任人、关联资源数、文档完整度。
- 顶部操作：新建项目、导入存量项目、查看统计。

#### 项目详情页

- 概览页签：项目基础信息、关键责任人、当前阶段、待办事项。
- 项目空间页签：标准目录树、非标准目录区、资料上传入口。
- 资源关联页签：IT资源、网络资源、数据资源、应用系统清单。
- 验收页签：验收申请、审核状态、整改意见、结项结论。

#### 模板管理页

- 左侧按项目类型和阶段展示模板分类树。
- 右侧展示模板目录结构和启用状态。
- 支持模板新增、复制、停用和版本管理。

#### 项目验收申请页

- 分块提交 IT管理、数据管理、安全管理、过程资料等材料。
- 每块展示必传项、当前缺失项、审核责任人。
- 底部展示总体验收流程和当前状态。

### 2.4 数据资源管理

#### 数据目录门户页

- 顶部为搜索框和常用标签。
- 左侧为分类导航：业务域、部门、数据类型、状态、安全等级。
- 中部卡片/列表展示资源名称、简介、更新频率、共享方式、状态。
- 支持查看详情、收藏、申请使用。

#### 数据资源详情页

- 基础信息标签页：名称、标识、简介、业务领域、责任人。
- 内容结构标签页：数据类型、格式、模式/字典。
- 治理信息标签页：血缘、质量规则、敏感标识、安全等级。
- 版本历史标签页：版本列表、快照、对比入口。
- 申请入口：发起数据使用申请或变更申请。

#### 数据资源维护/变更页

- 非核心属性编辑模式：直接保存，记录修改日志。
- 核心属性编辑模式：提交变更申请，进入审批流。
- 页面右侧显示历史版本摘要和审批状态。

### 2.5 网络安全管理

#### 网络资源台账页

- 筛选项：资源类型、IP 段、VLAN、责任人、安全状态、所属项目。
- 列表项：资源标识、IP/网段、带宽、策略摘要、使用状态。
- 顶部操作：新增台账、发起申请、导出文档。

#### 网络资源详情页

- 基础信息区：IP、VLAN、设备、端口、责任信息。
- 策略区：防火墙规则、访问控制列表、安全审核记录。
- 变更区：配置历史、变更记录、回收记录。
- 文档区：网络拓扑图、IP 分配表。

#### 网络申请/审核页

- 申请页按资源类型动态展示 IP、端口、带宽、访问方向等字段。
- 审核页展示需求摘要、安全校验项、配置计划、执行结果回填区。
- 执行后可直接写入台账和项目关联关系。

## 3. 关键交互流程

### 3.1 项目创建并装配模板

1. 用户从首页或项目列表点击“新建项目”。
2. 填写项目基础信息并选择项目类型。
3. 系统根据类型自动装配标准目录模板。
4. 用户确认责任人和管理团队信息后完成项目创建。
5. 跳转到项目详情页，默认打开“项目空间”页签。

### 3.2 IT资源申请到交付

1. 用户在项目详情或 IT资源页发起申请。
2. 选择资源类型后展示对应表单。
3. 提交后进入审批流，申请单状态显示为“待审核”。
4. 审批通过后生成“待交付”任务。
5. 运维人员回填部署信息并完成交付，系统自动生成资源台账。

### 3.3 应用登记并建立依赖图谱

1. 应用系统管理员新增应用。
2. 在应用详情中选择关联的项目、服务器、数据库、API。
3. 保存后生成依赖关系视图。
4. 当底层资源状态变化时，应用详情同步展示健康状态并推送消息。

### 3.4 数据资源申请与审批

1. 数据使用者在目录门户检索数据资源。
2. 进入详情页查看可申请性和共享条件。
3. 填写使用目的并提交申请。
4. 审批人处理申请，结果回写申请记录。
5. 申请通过后，详情页显示已授权状态或数据提供方式。

### 3.5 网络申请、审核与配置

1. 项目组成员发起网络申请。
2. 网络管理员在审核页校验 IP、端口、带宽和安全规则。
3. 审核通过后进入执行状态。
4. 配置完成后回填执行信息，更新网络台账并关联项目。

### 3.6 项目验收与结项

1. 项目经理发起验收申请。
2. 系统校验各验收分块资料是否齐备。
3. 审核人逐项处理并给出通过/退回意见。
4. 全部通过后项目状态切换为“已验收/已结项”。
5. 系统将后续待办推送给资源回收和资料归档责任人。

## 4. 页面状态设计

### 4.1 通用状态

- 空状态：首次无数据时提示如何新建项目、资源或数据。
- 加载状态：列表、详情、审批记录均提供骨架屏或加载占位。
- 成功状态：申请提交、审批完成、保存成功后明确提示结果和下一步入口。
- 权限受限状态：无权限时展示原因和可联系角色。

### 4.2 业务状态

- IT资源：待审核、待交付、使用中、变更中、待回收、已回收、异常。
- 应用：建设中、运行中、停用、异常。
- 项目：草稿、执行中、待验收、审核中、已验收、已结项。
- 数据资源：设计中、已上线、已下线、已归档。
- 网络资源：待审核、待配置、使用中、变更中、已回收、异常。

## 5. 异常流说明

- 申请被退回：保留原申请内容，突出退回原因，并支持基于原单据再次提交。
- 关联对象缺失：当项目、资源或应用已不存在或被停用时，禁止继续关联并给出提示。
- 外部监控数据缺失：详情页显示“暂无监控数据”，不阻塞基础台账查看。
- 验收材料不齐：验收页明确列出缺失材料、责任角色和补齐入口。
- 数据资源不可申请：若资源状态或权限不允许申请，应展示原因，例如“已归档”“权限不足”“需先完成审批前置”。
- 网络配置冲突：若检测到 IP 冲突或策略冲突，应在审核/执行页阻断提交并要求调整。
