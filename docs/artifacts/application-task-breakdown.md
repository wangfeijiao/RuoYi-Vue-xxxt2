## 1. 第一阶段范围

第一阶段必须落地以下能力：

- 扩展 `inf_application` 主档字段
- 引入项目关联关系
- 引入依赖关系管理
- 引入运行概览
- 引入异常事件与通知留痕
- 引入基础统计分析
- 保持现有 CRUD 可继续使用

以下内容明确放到后续阶段：

- 责任矩阵独立关系表
- 外部监控事件总线接入
- 短信/企业微信通知
- 依赖拓扑图
- 统计宽表

实现层应按“主档扩展 -> 关系治理 -> 状态联动 -> 异常通知 -> 统计分析”的顺序推进，而不是继续在现有单页表单和单个 Service 中堆叠逻辑。
